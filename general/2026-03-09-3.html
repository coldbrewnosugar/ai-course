Controlled Video Generation with LTX-2.3

From text prompts to structurally guided video — using depth maps, edge detection, and pose estimation to steer a diffusion model through ComfyUI
35 min video-generation comfyui lora diffusion creative-ai

What's happening

Lightricks has released LTX-2.3, the latest iteration of their open video generation model, alongside a significant architectural innovation: the Union IC-LoRA. Previous versions required separate LoRA adapters for each control condition — one for depth, another for edges, another for pose. The Union IC-LoRA collapses these into a single adapter that accepts multiple control signals simultaneously.

This matters for two reasons. First, it halves the memory overhead of multi-condition control. Second, it introduces downsampled latent processing, where control signals operate at reduced resolution before influencing the full-resolution generation. The result is faster inference without proportional quality loss — a trade-off that reflects a broader trend in diffusion model engineering toward efficiency without sacrificing controllability.

The model ships with a two-stage pipeline architecture: an initial generation pass followed by spatial and temporal upscaling. This separation of concerns — coarse structure first, fine detail second — mirrors established practices in image super-resolution and allows each stage to be independently tuned.

In this session, we use an AI agent to help us understand, configure, and reason about this pipeline. We will not write ComfyUI workflows by hand. Instead, we will develop the conceptual vocabulary to direct an agent through the setup, understand its outputs, and make informed decisions about model selection and control signal design.

1

Understand the Model Architecture and Its Components

Before touching any workflow, we need a clear mental model of what LTX-2.3 actually is and how its pieces fit together.

LTX-2.3 is a latent diffusion model for video. Like image diffusion models (Stable Diffusion, FLUX), it operates not on raw pixels but on compressed latent representations. The key difference is temporal: where an image model produces a single latent frame, a video model produces a sequence of latent frames that must be temporally coherent — objects should not teleport between frames.

The model comes in two variants:

  • Full model (ltx-2.3-22b-dev.safetensors): 22 billion parameters, higher fidelity, slower inference.
  • Distilled model (ltx-2.3-22b-distilled.safetensors): Same architecture, trained to approximate the full model's output in fewer denoising steps. Faster, but with some loss of fine detail.

The pipeline also requires several auxiliary models: a Gemma text encoder for prompt understanding, spatial and temporal upscalers for the second stage, and the Union IC-LoRA for control conditions. Each model has a specific directory it must be placed in — this is a common source of first-run errors.

The distilled model is not simply a smaller model — it is the same 22B architecture trained with knowledge distillation to converge in fewer steps. The quality difference is subtle at low step counts (4–8 steps) but becomes more pronounced when compared to the full model at 20+ steps.
What is latent diffusion and why does it matter for video?

Diffusion models generate content by iteratively removing noise from a random starting point. Operating directly on pixels is computationally prohibitive for video (a 5-second 720p clip at 24fps contains over 62 million pixels per frame, times 120 frames). Latent diffusion compresses each frame into a much smaller latent representation — typically 8× or 16× smaller in each spatial dimension — and performs the denoising there. The result is decoded back to pixel space only at the end.

For video, the latent space must also encode temporal relationships. LTX-2.3 uses a 3D latent structure where the temporal dimension is treated analogously to spatial dimensions, allowing the model to reason about motion and consistency across frames within the same compressed representation.

2

Design the Two-Stage Pipeline Architecture

The LTX-2.3 pipeline separates generation into two stages, and understanding this separation is essential to using it effectively.

Stage 1: Base Generation. The model produces a low-resolution video from a text prompt (and optionally a reference image). If using control conditions (depth maps, edge maps), the Union IC-LoRA modifies the denoising process at this stage. The output is a latent video at the model's native resolution.

Stage 2: Upscaling. The latent video is upscaled in two dimensions independently:

  • Spatial upscaling (1.5× or 2×) increases the resolution of each frame.
  • Temporal upscaling (2×) doubles the frame count by interpolating between existing frames, producing smoother motion.

These are not simple bicubic resizes. They are learned upscalers that operate in latent space, adding plausible detail rather than merely stretching existing information.

The two-stage design has a practical consequence: the first stage can use the distilled model for speed, while the second stage applies a distilled LoRA to maintain consistency. This means you can iterate quickly on composition and motion at low resolution, then commit to a slower upscaling pass only when satisfied.

Tip
The spatial and temporal upscalers are separate models because their tasks are fundamentally different. Spatial upscaling adds detail within a frame (textures, edges). Temporal upscaling synthesizes entirely new frames that must be motion-consistent with their neighbors. Separating them allows each to specialize.
Why upscale in latent space rather than pixel space?

Pixel-space upscaling (e.g., Real-ESRGAN) treats each frame as an independent image. It cannot reason about temporal consistency because it has no access to the model's internal representation of motion and structure. Latent-space upscaling, by contrast, operates on the same representation the model used during generation. This means the upscaler can add detail that is consistent with the model's understanding of the scene — a face turning will gain detail consistent with 3D geometry, not just sharpened edges. The trade-off is that latent upscalers are model-specific and cannot be reused across different architectures.

3

Understand the Union IC-LoRA and Control Signal Design

The Union IC-LoRA is the central innovation in this release, and understanding how it works changes how you think about controllable generation.

Traditionally, each control condition — depth, edges, pose — required its own adapter. Each adapter modified a specific subset of the model's weights, and combining them meant loading multiple adapters simultaneously, often with unpredictable interactions. The Union IC-LoRA is trained on all conditions jointly, so it learns a shared representation of structural guidance.

How control signals work mechanically: A control signal (say, a depth map) is an image the same dimensions as your target video frame. It is encoded and injected into the denoising process as additional conditioning — alongside the text prompt. The model learns to generate content that is consistent with both the text description and the structural constraints of the control signal.

Three control signals are supported:

  • Depth maps: Encode the relative distance of surfaces from the camera. They guide spatial layout — what is in front of what, how large objects are relative to each other.
  • Edge maps (Canny): Encode strong edges in a scene. They preserve structural boundaries — the outline of a building, the contour of a face.
  • Human pose: Encode skeletal keypoints. They guide body position and limb arrangement.

The Union LoRA also introduces downsampled latent processing: control signals are applied at a reduced resolution, then their influence propagates to the full resolution during denoising. This is not merely a performance optimization — it acts as a form of regularization, preventing the model from overfitting to pixel-level control signal details and instead capturing the structural intent.

Tip
You can extract depth and edge maps from existing video footage using standard computer vision tools (MiDaS for depth, OpenCV Canny for edges). This enables a powerful workflow: shoot rough reference footage, extract control signals, then generate stylized video that matches the original camera movement and composition.
What does 'downsampled latent processing' actually do?

When the Union IC-LoRA processes control signals at a reduced resolution, it is effectively blurring the control signal before applying it. Consider a depth map with fine detail — individual fingers at different depths, for instance. At full resolution, the model would try to match these precise depth boundaries, potentially creating artifacts where the diffusion process fights between the depth constraint and the text prompt.

At reduced resolution, the depth map becomes a smoother gradient: 'hand region is closer than background.' This gives the model more freedom to generate natural-looking fingers while respecting the overall spatial layout. The reduced resolution also means fewer cross-attention computations between the control signal and the latent, which directly reduces inference time. The approach is analogous to multi-scale processing in classical computer vision, where coarse features guide fine-scale generation.

4

Configure the Workflow and Reason About Parameters

With the conceptual foundation established, we now turn to the practical question of configuring the pipeline. The key parameters fall into three categories: model selection, sampling parameters, and control signal strength.

Model selection is the most consequential decision. The distilled model at 8 steps produces a usable result in roughly one-quarter the time of the full model at 30 steps. For iterative exploration — trying different prompts, adjusting control signals — the distilled model is the correct choice. Switch to the full model only for final renders where fidelity justifies the compute cost.

Sampling parameters include step count, CFG (classifier-free guidance) scale, and scheduler type. CFG scale controls how strongly the model follows the text prompt versus generating freely. Higher values (7–12) produce output more faithful to the prompt but can introduce artifacts; lower values (3–5) are more natural but may drift from the prompt. The distilled model typically works well with lower CFG because the distillation process already biases it toward prompt-following.

Control signal strength determines how much influence the depth/edge/pose conditioning exerts over generation. This is typically a scalar multiplier. At 1.0, the model treats the control signal as a hard constraint. At 0.5, it is a suggestion. At 0.0, it is ignored entirely. Finding the right balance is the core creative skill in controlled generation.

The agent should recommend a three-phase exploration strategy: (1) Fix control strength at 0.7 and CFG at 5, iterate on the text prompt using the distilled model at 8 steps until the subject matter and style are correct. (2) Hold the prompt fixed and sweep control strength from 0.3 to 1.0 in increments of 0.1, evaluating whether the output structure matches the control signal without artifacts (flickering edges, unnatural depth discontinuities). (3) Once composition is locked, switch to the full model and sweep CFG from 4 to 10, looking for the best balance between prompt fidelity and visual naturalness. The agent should note that upscaling parameters should only be tuned after the base generation is satisfactory, as upscaling amplifies both quality and defects.

Warning
Control signal strength interacts non-linearly with CFG scale. High control strength combined with high CFG can cause the model to 'fight' between two strong conditioning signals, producing visual artifacts — particularly at depth boundaries and object edges. If you see flickering or warping artifacts, reduce one or both values before adjusting other parameters.
Distilled vs. full model: when does the difference matter?

The distilled model approximates the full model's output distribution but does so by learning a shortcut through the denoising trajectory. At low step counts (4–8), this shortcut is remarkably effective — the distilled model produces coherent, well-composed video that is visually comparable to the full model at equivalent wall-clock time.

The difference becomes apparent in three specific areas: (1) fine texture detail, where the full model produces more varied and realistic surface textures; (2) temporal consistency in complex motion, where the full model maintains better coherence during rapid movement or occlusion; and (3) adherence to unusual prompts, where the full model's larger effective capacity handles edge cases better.

For most creative workflows, the practical recommendation is: use the distilled model for everything except final renders. The time saved during iteration far outweighs the marginal quality improvement of the full model.

5

Evaluate and Iterate on Generated Output

The final competence in agent-assisted video generation is evaluation — knowing what to look for in the output and how to diagnose problems.

Video generation introduces failure modes that do not exist in image generation. The most common:

Temporal flickering — frame-to-frame inconsistency in color, brightness, or texture. This often indicates that the control signal is too weak (the model has insufficient structural guidance to maintain consistency) or that CFG is too high (the model is over-responding to the prompt at the expense of temporal smoothness).

Depth bleeding — objects at different depths blending into each other, especially at boundaries. This is a sign that the depth map control is too strong at full resolution, causing hard depth boundaries that the diffusion process cannot reconcile with natural-looking transitions. Reducing control strength or relying on the downsampled latent processing (which naturally softens these boundaries) addresses this.

Motion drift — gradual deviation from the intended camera path or subject movement over the duration of the video. This is more common with longer generations and weaker control conditioning. Temporal upscaling can either mask or exacerbate this, depending on whether the drift is in the base generation or introduced by the upscaler.

Style collapse — the model defaulting to a generic aesthetic despite a specific style prompt. This often occurs when control signals are too strong, consuming the model's capacity for style interpretation. Reducing control strength typically restores stylistic range.

Tip
Always evaluate base generation output (Stage 1) before applying upscaling (Stage 2). Upscaling adds detail that can mask — or amplify — underlying problems. A flickering artifact in the base generation becomes a high-resolution flickering artifact after upscaling, which is harder to diagnose because it looks like a different problem.
How does temporal upscaling affect motion quality?

Temporal upscaling (2×) doubles the frame count by synthesizing intermediate frames. The upscaler is trained to interpolate motion, not merely blend pixels. This means it can produce plausible intermediate poses for a walking figure or smooth camera transitions.

However, it has limits. If the base generation contains a discontinuity — a sudden jump in camera position or an object appearing abruptly — the temporal upscaler will try to smooth over it, often producing a ghosting or morphing artifact that looks worse than the original discontinuity. The general rule: temporal upscaling improves good motion and worsens bad motion. Fix motion issues in Stage 1 before applying temporal upscaling in Stage 2.

Recap

This session examined the LTX-2.3 video generation pipeline through the lens of controlled generation — using structural signals to steer diffusion model output toward specific spatial and temporal goals.

We established a conceptual framework for the two-stage pipeline architecture, where base generation and upscaling are separated to enable efficient iteration. We examined the Union IC-LoRA's approach to unifying multiple control conditions into a single adapter with downsampled latent processing, understanding both why this improves efficiency and how it affects the balance between control precision and creative freedom.

The practical core of the session was learning to reason about parameter interactions — control strength, CFG scale, model variant, and step count — and to diagnose failure modes in generated output. These are skills that transfer across video generation systems, not just LTX-2.3.

Throughout, the agent served as a design partner: not executing workflows for us, but helping us structure our thinking about model selection, control signal design, and systematic parameter exploration.

  • The Union IC-LoRA unifies depth, edge, and pose control into a single adapter. Its downsampled latent processing trades pixel-level control precision for efficiency and regularization — a design choice that generally improves output quality by preventing over-constraining.
  • The two-stage pipeline (base generation + latent upscaling) enables an efficient workflow: iterate quickly with the distilled model at low resolution, then commit to upscaling only when the composition and motion are satisfactory.
  • Control signal strength and CFG scale interact non-linearly. Diagnosing artifacts requires isolating these parameters — always evaluate base generation before upscaling, and adjust one parameter at a time.
  • Selecting the right control signal is a creative decision, not a technical one. Depth maps preserve spatial layout; edge maps preserve structural boundaries; pose preserves body configuration. Each leaves different aspects unconstrained for the model and prompt to fill.
