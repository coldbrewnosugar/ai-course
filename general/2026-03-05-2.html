What If Text Generation Worked Like Image Generation?

Diffusion models changed images forever. Now they're coming for language — and you can convert any LLM into one.
40 min diffusion-lm dLLM LLaDA masked-diffusion model-training A2D

What's happening

You know how Stable Diffusion and DALL-E generate images? They start with pure noise and gradually refine it into a coherent picture. Every part of the image emerges in parallel — it's not like the model paints left to right, pixel by pixel.

Now here's what's wild: the dLLM library just hit GitHub Trending, and it lets you do the same thing with text. Instead of generating one token at a time (the way ChatGPT, Claude, and every other LLM works), diffusion language models start with a sequence of [MASK] tokens and iteratively unmask them — often many at once.

The headline feature? A2D (Autoregressive-to-Diffusion) — a recipe that takes any existing autoregressive model like Qwen 0.5B and converts it into a masked diffusion model. No pretraining from scratch. No massive compute budget. Just a finetuning recipe and a fundamentally different way to generate text.

This matters because diffusion LMs can:

  • Generate faster by filling in multiple tokens per step
  • Natively infill — fill in the middle of a document, not just append to the end
  • Revise globally — every token can influence every other token during generation

Let's get our hands dirty and see how this actually works.

1

Grok the Mental Model: Diffusion vs. Autoregressive

Before we touch any code, let's make sure we really get what's different here. This is the kind of understanding that'll make every prompt you write to your agent 10x better.

Autoregressive generation (GPT, Claude, LLaMA) works like writing a sentence on a typewriter. You type one letter, then the next, then the next. Each letter depends on everything before it, but you can never go back. The model literally predicts P(next token | all previous tokens).

Diffusion generation works like a sculptor. You start with a rough block (all [MASK] tokens), and each step reveals more detail. Crucially, the sculptor can work on the arms and legs at the same time — the model unmasks multiple positions in parallel.

Here's the key insight: in masked diffusion (the flavor dLLM uses), generation looks like this:

```

Step 0: [M] [M] [M] [M] [M] [M] [M] [M]

Step 1: [M] cat [M] [M] [M] [M] mat [M]

Step 2: The cat [M] [M] the [M] mat .

Step 3: The cat sat on the red mat .

```

The model doesn't go left-to-right. It fills in the tokens it's most confident about first, then refines. Think of it like doing a crossword puzzle — you fill in the easy clues first, and those help you figure out the hard ones.

What the agent gives back

The agent should explain that AR models are trained to predict the next token given all previous tokens (causal masking), while masked diffusion models are trained to predict ALL masked tokens simultaneously given the unmasked ones (like BERT on steroids). The key difference: AR models have a causal attention mask that enforces left-to-right ordering, while diffusion models use bidirectional attention — every position can see every other unmasked position. That's what enables parallel generation.

Tip
This bidirectional attention is exactly why BERT-style encoders can be turned into diffusion models too — they already see the full context. The dLLM library has a recipe for that (examples/bert).
Wait, if diffusion is so great, why isn't everyone using it?

Great question. Autoregressive models have a massive head start in scaling, training stability, and ecosystem support. Diffusion LMs are still catching up on quality — especially for long, coherent reasoning chains where left-to-right ordering gives you a natural "chain of thought." The dLLM paper shows competitive results on benchmarks, but AR models still win on tasks that need deep sequential reasoning. Think of it as: diffusion is better at drafting (fill everything in at once), AR is better at deliberating (think step by step). The interesting future might combine both.

3

Convert Qwen 0.5B to a Diffusion Model (A2D)

Here's where it gets really fun. The A2D recipe (Autoregressive-to-Diffusion) takes a pretrained autoregressive model and finetunes it to work as a masked diffusion model. The core idea is surprisingly elegant:

  1. Take Qwen 0.5B (a normal left-to-right model)
  2. Replace the causal attention mask with a bidirectional one
  3. Finetune on a masked diffusion objective — randomly mask some percentage of tokens, train the model to predict them all

That's it. The model already knows language from its AR pretraining. We're just teaching it a new generation strategy. It's like taking someone who learned to write essays (sequential) and teaching them to solve crossword puzzles (parallel fill-in) — the vocabulary and grammar knowledge transfers.

The dLLM repo has the recipe under examples/a2d/. The training uses HuggingFace's Trainer with LoRA (so we're not retraining all parameters), and it converges surprisingly fast — the repo reports good results with just a few hours on modest hardware.

Why does LoRA work so well here?

LoRA (Low-Rank Adaptation) only trains small adapter matrices while keeping the base model frozen. This works beautifully for A2D because the knowledge in the model (vocabulary, grammar, facts) doesn't need to change — we're only changing the generation mechanism (causal → bidirectional). The adapter learns the new attention pattern and masking strategy while the base weights preserve everything the model already knows. It's like teaching an old dog a new trick without making it forget the old ones.

Masked diffusion vs. block diffusion — what's the difference?

Masked diffusion (MDLM, used by LLaDA) unmasks individual tokens anywhere in the sequence. Block diffusion (BD3-LM) unmasks contiguous blocks of tokens left-to-right, which is a hybrid between AR and diffusion. Block diffusion tends to be more coherent for long sequences because it preserves some of the sequential structure, while pure masked diffusion is more flexible for infilling tasks. dLLM's A2D recipe supports both — you choose via the config.

Run Diffusion Inference — Watch Text Materialize

This is the moment of truth. We're going to generate text with our diffusion model and actually see the parallel unmasking in action. Unlike AR generation where tokens appear one at a time, diffusion generation looks almost magical — tokens pop into existence across the entire sequence simultaneously, with the model filling in the "easy" tokens first and gradually resolving the ambiguous ones.

dLLM provides an inference script that lets you watch this process step by step. The key parameters you need to think about:

  • Number of diffusion steps: More steps = higher quality but slower. Think of it like the "step count" slider in Stable Diffusion.
  • Sequence length: How many tokens to generate. Unlike AR models, you set this upfront (the model needs to know how many [MASK] tokens to start with).
  • Temperature / sampling strategy: Controls randomness, just like in AR models.

Honestly, the fixed sequence length thing confused me at first too. It feels weird to decide how long your output should be before generating it. But it makes sense — the model needs to allocate positions for all the masks. Some implementations pad with extra masks and trim, which helps.

Why do easy tokens get filled in first?

At each denoising step, the model predicts probabilities for all masked positions simultaneously. Positions where the model is highly confident (low entropy) get unmasked first. Function words like 'the', 'is', 'on' are usually filled in early because they're highly predictable from context. Content words like nouns and verbs come later because they carry more information and depend more on the full context. This is actually a beautiful emergent property — nobody programs this ordering, it falls out naturally from the confidence scores. It's similar to how humans process text: you can often guess the "skeleton" of a sentence before filling in the specific details.

5

Speed It Up with Fast-dLLM

So here's the practical question everyone asks: is diffusion generation actually faster than autoregressive?

The honest answer is: it depends. Vanilla diffusion with 64 steps might actually be slower than AR generation because each step requires a full forward pass through the model. But Fast-dLLM changes the math with two clever tricks:

  1. KV-Cache for diffusion: Just like AR models cache key-value pairs to avoid recomputing attention for previous tokens, Fast-dLLM caches computations for positions that are already unmasked. Since those tokens don't change between steps, we don't need to recompute them.
  1. Confidence-threshold decoding: Instead of using a fixed number of steps, the model adaptively decides when to unmask each position. If the model is 99% confident about a token, unmask it immediately. If it's uncertain, keep it masked for more steps. This means easy generations finish in fewer steps.

The combination can give 2-8x speedups over vanilla diffusion, and for short generations, it can actually beat AR generation in wall-clock time. The examples/fastdllm/ directory has everything we need.

Tip

Warning
Benchmarking on a single prompt is misleading. Diffusion models have high variance in generation time because confidence-threshold decoding makes some prompts fast and others slow. Use at least 50 diverse prompts for meaningful numbers.
How does confidence-threshold decoding actually work under the hood?

At each diffusion step, the model outputs a probability distribution over the vocabulary for every masked position. The confidence score is simply the max probability — if the model assigns 95% probability to a single token, it's very confident. Confidence-threshold decoding unmasks all positions where max_prob > threshold in a single step, regardless of the scheduled masking ratio. This means the model can unmask 50 tokens in one step if it's confident about all of them, or only 2 tokens if the context is ambiguous. The adaptive nature is what gives the speedup — you're not wasting compute on positions the model has already figured out.

Recap

We just explored a fundamentally different paradigm for text generation. Instead of the one-token-at-a-time approach that every major LLM uses, diffusion language models generate text the way Stable Diffusion generates images — starting from noise (masks) and iteratively refining in parallel.

Using the dLLM library, we:

  • Understood why diffusion works for text (bidirectional attention + parallel unmasking)
  • Set up the environment and used the A2D recipe to convert a standard Qwen 0.5B model into a masked diffusion model
  • Watched text materialize through the denoising process — seeing the model fill in confident tokens first
  • Benchmarked Fast-dLLM's confidence-threshold decoding for real speedups

The big picture? Autoregressive generation won't be the only game in town forever. Diffusion models offer native infilling, parallel generation, and a fundamentally different quality-speed tradeoff curve. They're not replacing GPT-style models tomorrow, but the gap is closing fast — and tools like dLLM make it easy to experiment yourself.

  • Diffusion LMs generate text by iteratively unmasking tokens in parallel, filling in confident positions first — like solving a crossword puzzle rather than writing left to right
  • The A2D recipe can convert ANY autoregressive model into a diffusion model via LoRA finetuning, reusing the model's existing knowledge while teaching it a new generation strategy
  • Fast-dLLM's confidence-threshold decoding adaptively decides when to unmask each token, giving 2-8x speedups over vanilla diffusion and making it competitive with AR generation speed
  • Diffusion models natively excel at infilling tasks (filling in the middle) because they use bidirectional attention — something AR models can only approximate with tricks
  • Even BERT-family encoders can be converted into generative chat models through diffusion, blurring the line between encoder and decoder architectures

Where to go next

  • Try the block diffusion variant (BD3-LM) in dLLM — it generates contiguous blocks left-to-right, a hybrid between AR and diffusion that may give better coherence on long texts
  • Explore the Edit Flows recipe (examples/editflow/) for teaching diffusion models to perform structured edits — insertion, deletion, and substitution — on existing text
  • Run the lm-evaluation-harness benchmarks to quantitatively compare your A2D model against the original AR model on reasoning and knowledge tasks
  • Experiment with different masking schedules (linear, cosine, sqrt) during training and inference — the schedule dramatically affects generation quality and is an active research area
